# .github

이 저장소는 조직의 GitHub 설정 및 템플릿을 관리합니다.

## 📋 Issue Templates

### ✨ Task (할 일/개발)
실제 코드 작성이나 설정 작업을 정의할 때 사용합니다.
- 작업 내용 (What): 무엇을 할 것인지 명확히 기술
- 기술적 계획 (How): 구체적인 구현 단계를 체크리스트로 작성
- 체크리스트: 완료 조건 확인

### 🚀 Epic (기획)
큰 기능 단위의 목표와 전반적인 스펙을 정의할 때 사용합니다.
- Client, Server, AI 영역별로 구현 범위 정리
- 하위 Task들을 포함하는 상위 단위 이슈

## 📝 Pull Request Template
PR 생성 시 자동으로 적용되는 템플릿입니다.
- 관련 이슈 링크
- 작업 내용 요약
- 의존성 및 주의사항
- 자가 점검 체크리스트
